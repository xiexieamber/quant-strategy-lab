# 量化交易入门指南

## 什么是量化交易？

**量化交易** = 用**数据和规则**来做买卖决策，而不是靠感觉。

简单比喻：

- **主观交易**：「我觉得苹果要涨，买！」
- **量化交易**：「当 5 日均线超过 20 日均线时，系统自动买入」

## 量化策略开发的 4 个步骤

```
1. 获取数据  →  2. 定义策略  →  3. 回测验证  →  4. 分析优化
```

### 第 1 步：获取数据

需要历史价格数据，常见字段叫 **OHLCV**：


| 字段     | 含义  |
| ------ | --- |
| Open   | 开盘价 |
| High   | 最高价 |
| Low    | 最低价 |
| Close  | 收盘价 |
| Volume | 成交量 |


本项目用 `yfinance` 免费获取数据，见 `src/data/fetch_data.py`。

### 第 2 步：定义策略

策略就是一套**明确的买卖规则**。例如双均线：

- 短期均线 > 长期均线 → 买入（持有）
- 短期均线 < 长期均线 → 卖出（空仓）

代码在 `src/strategies/dual_moving_average.py`。

### 第 3 步：回测（Backtest）

**回测** = 用历史数据模拟「如果当时按这个策略交易，结果会怎样」。

重要概念：

- **未来函数**：不能用「今天才知道的信息」做「今天的决策」（这是作弊）
- **初始资金**：假设一开始有多少钱
- **全仓**：简化起见，要么全买要么全卖

回测引擎在 `src/backtest/engine.py`。

### 第 4 步：分析结果

回测完成后，看这些指标：


| 指标   | 含义               | 怎么看             |
| ---- | ---------------- | --------------- |
| 总收益率 | 整个期间赚/亏了多少       | 越高越好，但要结合风险     |
| 最大回撤 | 从最高点最多跌了多少       | 越小越好（负数，如 -15%） |
| 夏普比率 | 每承担一份风险，得到多少超额收益 | 一般 >1 算不错       |
| 交易次数 | 买卖切换了多少次         | 太频繁可能手续费吃掉利润    |


## 新手常见误区

1. **回测赚钱 ≠ 实盘赚钱**
  历史表现好，未来不一定好。这叫做「过拟合」风险。
2. **忽略手续费和滑点**
  本项目为了简化没算手续费，实盘会有成本。
3. **策略越复杂越好？**
  不对。简单策略更容易理解和维护，复杂策略更容易过拟合。
4. **只看不做的学习**
  一定要自己改参数、跑回测、看结果变化，才能真的学会。
