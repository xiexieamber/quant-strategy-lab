# 量化交易入门指南

> 本文部分内容整理自聚宽（JoinQuant）社区经典教程 [《初识量化交易》](https://www.joinquant.com/view/community/detail/3ca1564dd4831740cc4d0b5a3c01a957)，并结合本项目的代码结构做了对照说明，方便零基础学习。

## 什么是量化交易？

**量化交易** = 借助数学、统计和计算机，用**数据和规则**来做买卖决策，而不是靠感觉。

简单比喻：

- **主观交易**：「我觉得苹果要涨，买！」——像用锄头镰刀手工收割
- **量化交易**：「当 5 日均线超过 20 日均线时，系统自动买入」——像用大型收割机批量处理

在美国等成熟市场，量化交易已占股市成交量 70% 以上。它的核心优势是：**效率更高、判断更客观、能同时监控全市场**。

### 量化交易到底在做什么？

聚宽教程把量化开发拆成 4 个环节，和本项目的流程一一对应：

```
灵感（想法） → 细化策略（写清楚规则） → 转成程序（写代码） → 检验效果（回测/模拟）
```

| 环节 | 通俗解释 | 在本项目里 |
| ---- | -------- | ---------- |
| 灵感 | 「股价低于均线就买」这类模糊想法 | 你自己想策略逻辑 |
| 细化策略 | 把模糊想法写成任何人都能执行的明确规则 | 写 `config.yaml` 参数 + 策略函数 |
| 转成程序 | 让计算机能读懂并执行 | `src/strategies/` 里的 Python 代码 |
| 检验效果 | 用历史或实时数据验证好不好 | `scripts/run_backtest.py` 回测 |

## 从「灵感」到「可执行策略」：一个完整例子

聚宽教程用一个均线例子，演示如何把模糊想法变成明确规则。理解这个过程，比背代码更重要。

### 第 0 步：灵感（还很模糊）

```
如果股价显著低于近几日的平均价，则买入
如果股价显著高于近几日的平均价，则卖出
```

问题：「显著低于」是多少？「近几日」是几天？买多少？卖哪只股票？

### 第 1 步：细化成明确规则

```
如果股价低于近 20 日平均价 10%，则用全部可用资金买入
如果股价高于近 20 日平均价 10%，则卖出全部所持的该股票
```

再补充：监控除 ST 股外的 A 股；每个交易日检查一次。

**关键原则**：策略必须足够明确，让不同人在相同情况下做出相同操作。不必一次做到完美，可以边回测边改。

### 第 2 步：对应到本项目

本项目的双均线策略是另一个经典例子，逻辑更简单：

- 短期均线 > 长期均线 → 买入（持有）
- 短期均线 < 长期均线 → 卖出（空仓）

参数在 `strategies/dual_ma/config.yaml`，逻辑在 `src/strategies/dual_moving_average.py`。

## 量化策略开发的 4 个步骤（本项目版）

```
1. 获取数据  →  2. 定义策略  →  3. 回测验证  →  4. 分析优化
```

### 第 1 步：获取数据

需要历史价格数据，常见字段叫 **OHLCV**：

| 字段     | 含义  |
| ------ | --- |
| Open   | 开盘价 |
| High   | 最高价 |
| Low    | 最低价 |
| Close  | 收盘价 |
| Volume | 成交量 |

本项目支持两种数据源：

| 数据源 | 适合 | 是否需要账号 | 代码位置 |
| ------ | ---- | ------------ | -------- |
| **yfinance** | 美股（AAPL、MSFT 等） | 不需要 | `src/data/fetch_data.py` |
| **聚宽 JQData** | A 股真实数据（推荐） | 需要注册 + 申请试用 | `src/data/fetch_jqdata.py` |

> 做 A 股策略建议用聚宽 JQData，数据更准、字段更全；yfinance 的 A 股数据质量不稳定。

### 第 2 步：定义策略

策略就是一套**明确的买卖规则**。聚宽平台的典型代码结构是：

```python
def initialize(context):   # 初始化：设置股票池、参数等
    g.security = ['002043.XSHE', '002582.XSHE']

def handle_data(context, data):  # 每个周期执行：读数据、判断、下单
    for stock in g.security:
        last_price = data[stock].close
        average_price = data[stock].mavg(20, 'close')
        # ... 根据规则买卖
```

本项目不直接用聚宽 API，但思路一样：**读数据 → 算指标 → 产生信号（1=持有，0=空仓）**。

### 第 3 步：回测（Backtest）

**回测** = 用**历史数据**模拟「如果当时按这个策略交易，结果会怎样」。

聚宽教程的设定示例：

- 初始资金：500,000 元
- 回测区间：2006-01-01 到 2016-01-01
- 输出：盈亏、下单记录、持仓变化、年化收益、最大回撤、夏普比率等

重要概念：

- **未来函数**：不能用「今天才知道的信息」做「今天的决策」（这是作弊）
- **初始资金**：假设一开始有多少钱
- **全仓**：简化起见，要么全买要么全卖

回测引擎在 `src/backtest/engine.py`。

#### 回测 vs 模拟交易

| 方式 | 用什么数据 | 用途 |
| ---- | ---------- | ---- |
| **回测** | 历史数据 | 快速验证策略在过去是否有效 |
| **模拟交易** | 实时行情 | 用「未来」的真实数据进一步验证 |

建议顺序：**先回测 → 结果不错再模拟 → 最后才考虑实盘**。

### 第 4 步：分析结果

回测完成后，看这些指标（聚宽教程重点强调的三个）：

| 指标   | 含义               | 怎么看             |
| ---- | ---------------- | --------------- |
| 总收益率 / 年化收益率 | 整个期间赚/亏了多少       | 越高越好，但要结合风险     |
| 最大回撤 | 从最高点最多跌了多少       | 越小越好（负数，如 -15%） |
| 夏普比率 | 每承担一份风险，得到多少超额收益 | 一般 >1 算不错       |
| 交易次数 | 买卖切换了多少次         | 太频繁可能手续费吃掉利润    |

## 为什么值得学量化？（4 个核心价值）

来自聚宽教程的总结，帮你理解「为什么要花时间学这个」：

1. **效率提升百倍**：人工验证 10 个交易日不够，计算机几分钟可以回测上千天
2. **评价更客观**：用数学指标（夏普、回撤等）代替「感觉不错」
3. **全市场盯盘**：几千只股票同时监控，不错过满足条件的时机
4. **发现隐藏规律**：比如「胜率 55%、盈亏相等」这类需要大量交易才能验证的规律，只有量化能做到

## 新手常见误区

1. **回测赚钱 ≠ 实盘赚钱**
   历史表现好，未来不一定好。这叫做「过拟合」风险。
2. **忽略手续费和滑点**
   本项目为了简化没算手续费，实盘会有成本。
3. **策略越复杂越好？**
   不对。简单策略更容易理解和维护，复杂策略更容易过拟合。
4. **只看不做的学习**
   一定要自己改参数、跑回测、看结果变化，才能真的学会。
5. **跳着读教程**
   聚宽系列前后连贯，建议按目录顺序学；遇到不懂的先记下来，后面章节可能会讲到。

## 用聚宽 API 获取真实 A 股数据

聚宽有**两套 API**，初学者容易混淆：

| 名称 | 在哪里用 | 能不能在本项目本地用 |
| ---- | -------- | -------------------- |
| **官网 API**（jqdata） | 聚宽网站上的策略编辑器、在线 Notebook | ❌ 只能在聚宽网页里用 |
| **JQData SDK**（jqdatasdk） | 你自己电脑的 Python 项目 | ✅ 就是你要用的这个 |

### 第 1 步：注册并申请试用

1. 打开 [聚宽官网](https://www.joinquant.com/) 注册账号
2. 申请 [JQData 试用](https://www.joinquant.com/default/index/sdk)（新用户通常有 1 年免费额度）
3. 密码一般是手机号后 6 位（以官网说明为准）

### 第 2 步：安装 SDK

```bash
python3 -m pip install jqdatasdk
```

### 第 3 步：配置账号（不要写进代码）

```bash
# 复制模板并填写
cp .env.example .env

# 或在终端临时设置
export JQ_USERNAME="你的手机号"
export JQ_PASSWORD="你的聚宽密码"
```

### 第 4 步：拉取数据

**A 股代码格式**（和 yfinance 不同）：

| 股票 | 聚宽代码 | yfinance 代码 |
| ---- | -------- | ------------- |
| 平安银行 | `000001.XSHE` | `000001.SZ` |
| 贵州茅台 | `600519.XSHG` | `600519.SS` |
| 沪深300指数 | `000300.XSHG` | — |

`.XSHE` = 深交所，`.XSHG` = 上交所。

**最小示例**（3 行核心代码）：

```python
import jqdatasdk as jq

jq.auth("手机号", "密码")
df = jq.get_price("000001.XSHE", start_date="2023-01-01", end_date="2023-12-31")
print(df.head())
```

**在本项目里运行演示**：

```bash
python scripts/fetch_jqdata_demo.py
```

数据会保存到 `data/raw/` 目录。

### 常用 API 速查

| 函数 | 作用 | 示例 |
| ---- | ---- | ---- |
| `auth()` | 登录 | `jq.auth("手机号", "密码")` |
| `get_price()` | 历史行情 | `jq.get_price("600519.XSHG", start_date="2020-01-01")` |
| `get_all_securities()` | 全部股票列表 | `jq.get_all_securities(types=["stock"])` |
| `get_index_stocks()` | 指数成分股 | `jq.get_index_stocks("000300.XSHG")` |
| `get_fundamentals()` | 财务数据 | 需配合 `query()` 使用 |

完整文档：[JQData 说明书](https://www.joinquant.com/help/api/help?name=JQData)

> **本项目 AI 速查**：详见 [docs/jqdata-api-reference.md](jqdata-api-reference.md)（含常见错误、代码模板、权限说明）
