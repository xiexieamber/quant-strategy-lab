# 第一轮策略审计

审计日期：2026-06-08

本金假设：100000

## 结论

当前项目已经有较好的学习和小市值实验基础，但还不能直接交给 Cursor 做“全自动调参优化”。原因不是代码太差，而是缺少标准实验记录、样本切分、反过拟合审计和纸交易闭环。

建议优先优化对象是小市值 H1-Pro 流程，而不是双均线入门示例。

## 当前结构

| 模块 | 文件 | 审计结论 |
| --- | --- | --- |
| 双均线策略 | `src/strategies/dual_moving_average.py` | 适合教学，不适合作为真实资金优化对象 |
| 入门回测 | `src/backtest/engine.py` | 明确不含手续费和滑点，适合学习原理 |
| 小市值配置 | `src/strategies/small_cap/config.py` | 已有 H1-Pro 预设，初始资金可设为 100000 |
| 小市值回测 | `src/backtest/small_cap_engine.py` | 已建模交易成本、涨跌停、停牌、指数择时、持仓日志 |
| 果仁批量回测 | `scripts/guorn_batch_backtest.py` | 可用于批量验证，但需避免只按单次综合分调参 |

## 已有优点

1. 小市值策略已有独立配置对象 `SmallCapConfig`。
2. H1-Pro 预设已默认使用 100000 初始资金。
3. 小市值引擎使用前一日指数择时，降低明显未来函数风险。
4. 已有 ST、北交所、科创板、盈利、上市天数、成交额、差审计意见、退市整理等过滤字段。
5. 已有交易成本、涨跌停限制、停牌处理、黄金 ETF 空仓替代和持仓日志。
6. 果仁批量回测脚本会保存批量结果 JSON。

## 主要风险

### P1：缺少标准实验记录

目前小市值本地回测返回 `SmallCapBacktestResult`，但没有统一保存：
- config
- metrics
- yearly_returns
- holdings_log
- split 信息
- 代码版本
- 实验报告

这会导致 Cursor 多轮优化后难以复现“哪次为什么变好”。

### P1：缺少样本外验证

项目里已有参数组合文件和综合评分，但没有强制 train/validation/test 或 walk-forward。自动优化如果直接看全周期结果，很容易过拟合。

### P1：优化目标还不够稳健

`src/guorn/metrics.py` 的综合分包含年化、夏普、回撤、2024 年收益，这是好的开始，但仍缺少：
- 参数邻域稳定性
- 交易次数下限
- 年度收益均衡性
- 成本/滑点压力测试
- 与基准的超额收益评价

### P2：教学回测引擎容易被误用

`src/backtest/engine.py` 注释明确说明不考虑手续费和滑点。如果 Cursor 直接优化双均线，它可能给出“看起来很好”的结果，但实盘参考价值很低。

### P2：本地小市值引擎还没有滑点模型

小市值引擎有 `trade_cost`，但没有单独滑点/冲击成本模型。对小市值策略来说，成交价格假设很重要。

### P2：缺少纸交易闭环

目前没有每日信号、模拟成交、未成交原因、回测差异的纸交易记录。直接实盘风险偏高。

## 建议修改顺序

1. 新增实验运行器，保存 `results/experiments/...`。
2. 为小市值回测增加 baseline 命令行入口。
3. 增加 train/validation/test 配置。
4. 增加参数搜索脚本，但默认只跑小范围。
5. 增加压力测试：更高手续费、延迟一天调仓、不同起止日期。
6. 增加纸交易日志格式。
7. 最后再让 Optimization Agent 执行自动调参。

## 第一轮禁止事项

在完成实验框架前，不建议：
- 大范围搜索参数。
- 改策略逻辑追求更高收益。
- 删除亏损实验结果。
- 只使用全周期收益做评价。
- 用真实资金执行信号。

## 0 交易问题修复记录

修复日期：2026-06-08

根因：

1. 旧 AkShare 股票缓存里包含当前环境无法反序列化的 pandas/pyarrow dtype，`pickle.load()` 抛出 `ModuleNotFoundError: No module named 'pyarrow'`。
2. 并行下载层吞掉了单股异常，导致全市场股票都被视为下载失败，最终 `valuation_store=0`、`prices=0`。
3. AkShare 路径里部分接口默认重试过久，诊断和实验容易表现为长时间无输出。
4. 北交所过滤只排除了 `8` 开头，没有排除 `4`、`9` 开头，导致 `920xxx` 标的进入持仓。

已修复：

- 坏缓存读取失败时自动重新下载并覆盖缓存。
- 全市场市值、候选股 K 线、个股 meta 下载改为轻量重试。
- `build_ak_avg_money_maps()` 改为向量化 rolling 计算。
- AkShare 路径移除重复实时 ST 标记查询，复用 `get_universe_codes(exclude_st=True)` 的预过滤结果。
- 北交所过滤改为排除 `8/4/9` 开头代码。
- 新增 `scripts/diagnose_small_cap_funnel.py` 输出过滤漏斗。
- 新增测试覆盖坏缓存刷新和北交所过滤。
