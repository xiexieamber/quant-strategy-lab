# 策略开发流程

本文说明在本项目中，如何从零添加一个新策略。

## 标准流程

```
config.yaml  →  strategy.py  →  run_backtest.py  →  看结果  →  优化
   配置参数        编写逻辑         运行回测          分析指标     迭代
```

## 以双均线为例（已有）

### 1. 配置文件

位置：`strategies/dual_ma/config.yaml`

这里放**和策略相关的参数**，不写代码逻辑：

```yaml
symbol: AAPL
short_window: 5
long_window: 20
```

### 2. 策略逻辑

位置：`src/strategies/dual_moving_average.py`

函数输入价格，输出买卖信号（1=持有，0=空仓）。

### 3. 运行脚本

位置：`scripts/run_backtest.py`

串联：读配置 → 下载数据 → 算信号 → 回测 → 打印结果。

## 如何添加自己的策略

假设你要做一个「RSI 超卖买入、超买卖出」策略：

### 步骤 1：新建配置目录

```
strategies/rsi/
└── config.yaml
```

### 步骤 2：编写策略函数

在 `src/strategies/rsi_strategy.py` 中：

```python
def rsi_signals(prices, period=14, oversold=30, overbought=70):
    # 计算 RSI
    # 返回 signal 列
    ...
```

### 步骤 3：编写运行脚本

复制 `scripts/run_backtest.py`，改成调用你的策略函数。

### 步骤 4：运行并分析

```bash
python scripts/run_rsi_backtest.py
```

## 回测结果怎么判断「好不好」？

没有绝对标准，但可以用这个 checklist：

- [ ] 总收益是否为正？
- [ ] 最大回撤能否接受？（例如不超过 -20%）
- [ ] 和「买入持有」比，策略是否更好？
- [ ] 换一只股票/换一个时间段，结果是否还合理？
- [ ] 交易次数是否过于频繁？

最后一项很重要：**如果换参数、换股票结果差很多，可能是过拟合**。

## 推荐学习顺序

1. 跑通双均线，理解全流程
2. 改 `short_window` / `long_window`，观察结果变化
3. 换 `symbol`（如 MSFT、SPY），对比不同标的
4. 自己实现 RSI 或 MACD 策略
5. 在 `notebooks/` 里用 Jupyter 做可视化分析

## 进阶方向（以后可以学）

- 加入手续费、滑点模拟
- 多标的组合（Portfolio）
- 使用 Backtrader、Zipline 等专业回测框架
- 因子研究（价值、动量、质量等）
- 机器学习预测（需谨慎，容易过拟合）
