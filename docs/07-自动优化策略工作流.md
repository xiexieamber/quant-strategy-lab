# 自动优化策略工作流

本文档给 A 总和 Cursor/Agent 使用。目标是让 agent 自动推进策略优化，但不把任务变成“刷最高回测收益”。

## 总原则

1. 先审计，后优化。
2. 先保证回测可信，再谈收益。
3. 先样本外稳定，再追求单次高收益。
4. 所有实验必须可复现、可回滚、可比较。
5. 本金按 100000 设计风控，第一阶段不使用杠杆。

## 推荐 Agent 分工

| Agent | 任务 | 可否改代码 | 交付物 |
| --- | --- | --- | --- |
| Controller Agent | 调度全流程，决定下一步 | 可以 | 本轮执行计划、验收清单 |
| Research Agent | 解释策略收益来源 | 不建议 | 策略假设、失败场景 |
| Data Quality Agent | 检查数据质量和未来函数 | 可以补检查 | 数据审计报告 |
| Backtest Engine Agent | 改进回测和实验记录 | 可以 | 可复现实验输出 |
| Optimization Agent | 参数搜索和 walk-forward | 可以 | 参数结果表 |
| Overfit Audit Agent | 找过拟合和脆弱点 | 不建议 | 反过拟合 verdict |
| Risk Agent | 设计 10 万本金风控 | 可以补配置 | 仓位、止损、熔断规则 |
| Paper Trading Agent | 模拟盘落地 | 可以 | 每日信号和执行记录 |

## 当前项目定位

- `src/backtest/engine.py`：教学用双均线向量化回测，不含手续费/滑点，不适合作为真实优化目标。
- `src/backtest/small_cap_engine.py`：更接近真实研究对象，已包含交易成本、开盘/收盘调仓、停牌、涨跌停限制、指数择时、ST/北交所/科创板等过滤。
- `scripts/guorn_batch_backtest.py`：用于果仁网页接口批量调参，适合小批量验证，不应高频请求或只按单次收益排名。
- `apps/small_cap_lab/app.py`：手动实验入口，适合观察单次参数组合效果。

## 推荐自动优化流程

### 第 1 阶段：项目审计

目标：明确现在能不能信任回测。

必须检查：
- 数据是否按交易日对齐。
- 信号是否使用未来数据。
- 买卖价格是否符合实际可成交假设。
- 手续费、滑点、停牌、涨跌停是否纳入。
- 回测指标是否包含回撤和年度表现。
- 参数结果是否保存到 append-only 文件。

输出：
- `docs/08-第一轮策略审计.md`
- 待修问题清单

### 第 2 阶段：实验记录标准化

每次实验至少保存：
- strategy_name
- config
- data_source
- start/end
- split_name
- metrics
- yearly_returns
- holdings/trades summary
- created_at
- code_version，如果可获取 git commit

建议输出目录：

```text
results/experiments/
  YYYYMMDD-HHMMSS-small-cap-baseline/
    config.json
    metrics.json
    yearly_returns.csv
    holdings_log.csv
    report.md
```

### 第 3 阶段：基线回测

先固定一组基线：

```python
SmallCapConfig.for_h1_pro(
    initial_cash=100_000,
    start="2021-01-01",
    end=None,
)
```

如果数据源限制导致无法跑长周期，记录限制原因，不能隐瞒。

### 第 4 阶段：样本切分

建议至少使用：
- train：2021-01-01 到 2023-12-31
- validation：2024-01-01 到 2024-12-31
- test：2025-01-01 到最近完整交易日

如果当前数据不足，使用 walk-forward：
- 每 12 个月训练
- 后 3 个月验证
- 滚动推进

### 第 5 阶段：参数优化

只允许小范围、可解释参数：
- `buy_rank`
- `sell_rank`
- `max_positions`
- `circ_weight`
- `total_weight`
- `empty_months`
- `stop_loss_pct`
- `index_ma`
- `trade_cost` 只用于压力测试，不应作为优化对象

禁止：
- 看到 test 结果后继续调参。
- 只保留赚钱组合。
- 以单一年化收益做唯一目标。

建议评分：

```text
score =
  30% 样本外年化收益排名
+ 25% 样本外 Sharpe 排名
+ 25% 最大回撤排名
+ 10% 参数邻域稳定性
+ 10% 年度收益均衡性
```

### 第 6 阶段：反过拟合审计

必须回答：
- 最优参数附近是否也表现良好。
- 收益是否来自少数几笔交易。
- 2024/2025 是否单独失效。
- 交易成本提高 50% 后是否仍可接受。
- 调仓延迟一天后是否仍可接受。
- 跟基准相比是否真的有超额收益。

### 第 7 阶段：10 万本金风控

默认规则：
- 单笔风险不超过 500 到 1000。
- 单策略最大回撤预警 8%，强制复盘 12%。
- 单标的目标仓位不超过 10% 到 20%。
- 第一阶段仅纸交易，不上杠杆。
- 连续 3 次数据异常或信号无法成交，暂停策略。

### 第 8 阶段：纸交易

纸交易至少 4 到 8 周，记录：
- 每日信号。
- 目标持仓。
- 模拟成交价。
- 未成交原因。
- 与回测假设差异。
- 当日净值和回撤。

```text
第一步只允许审计和补充实验框架，不允许直接调参刷收益。

请先输出：
1. 当前策略是否适合自动优化
2. 当前回测可信度评分 0-100
3. 数据质量风险
4. 回测引擎风险
5. 最小可行实验框架设计
6. 准备修改的文件
7. 验证命令

然后执行最小修改，使项目能保存可复现实验结果。
```
